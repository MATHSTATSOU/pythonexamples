Configuring Python for the pythonexamples Package

Wayne S. Stewart

2026-04-24

Introduction

The pythonexamples package demonstrates how to integrate Python code into an R package using the reticulate framework.

This vignette explains:

  • how Python is selected and managed by reticulate,
  • how required Python packages are installed,
  • why session order matters,
  • and how to verify that the configuration is correct.

No prior experience with Python is required.

Why reticulate?

The reticulate package provides a robust and reproducible interface between R and Python. It allows R users to:

  • call Python functions from R,
  • import Python modules,
  • manage Python environments,
  • and embed Python code inside R packages.

The pythonexamples package uses reticulate according to recommended best practices for package development.

How pythonexamples Uses Python

The package includes Python source files stored under:

inst/python/

For example:

inst/python/power.py

These Python files implement numerical routines that are called from R using reticulate.

The required Python components are:

  • Python ≥ 3.9
  • NumPy
  • SciPy

Automatic Python Configuration

When the pythonexamples package is loaded, it declares its Python requirements automatically.

This is done using the .onLoad() mechanism in the file:

R/zzz.R

with the following code:

.onLoad <- function(libname, pkgname) {
  reticulate::py_require(c("numpy", "scipy"))
}

This ensures that:

  • Python is configured before any Python code runs
  • Required Python packages are available
  • Users do not need to manually manage Python environments

Installing Python Dependencies Manually (If Needed)

In rare cases (for example, restricted systems or fresh installations), dependencies may not be installed automatically.

To install them manually, run the following before any Python code is executed:

reticulate::py_require(c("numpy", "scipy"))

Important:
If Python has already been initialized in the session, restart R before running this command.

Session Order Matters

Python is initialized only once per R session.

Correct usage:

library(pythonexamples)
one_sample_power(delta = 0.5, sigma = 1, n = 25)
#> [1] 0.705418

Incorrect usage:

reticulate::source_python("inst/python/power.py")
library(pythonexamples)

If Python is initialized before requirements are declared, dependencies cannot be added until the session is restarted.

Verifying That Everything Works

To confirm that Python integration is working correctly:

one_sample_power(delta = 0.5, sigma = 1, n = 25)
#> [1] 0.705418

You should receive a numeric power value without errors.

Troubleshooting

SciPy not found

If you see an error such as:

ModuleNotFoundError: No module named 'scipy'

take the following steps:

  1. Restart R
  2. Run:
reticulate::py_require(c("numpy", "scipy"))
  1. Reload the package:
library(pythonexamples)

The pythonexamples package relies on reticulate’s managed Python environments and does not require system-wide Python installation.

Summary

  • pythonexamples uses reticulate in its recommended configuration
  • Python dependencies are declared automatically on package load
  • Most users do not need to manage Python manually
  • Session order matters for successful Python integration
  • reticulate::py_config() is the primary diagnostic tool

This approach is robust, reproducible, and suitable for teaching, research, and package development.
